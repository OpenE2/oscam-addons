loghistorylines

parameter is optional

loghistorylines = lines

Size of log message history in web interface or monitor.

Example:

loghistorylines	= 0 	# Live Log and Log in status page is disabled
loghistorylines	= 64 	# Minimum value for Live Log and Log in status page
loghistorylines	= 9999 	# Maximum value for Log in status page - Live Log have limit 512

Default:

loghistorylines	= 256

This is how it works in Log in status page:

Without Limit

This is how it works in Live Log:

Live Log have parameter "maxloglines = 512". If you set "loghistorylines > 512" than comes the limit.

Oscam saves the last "loghistorylines" log lines in a buffer. Your browser gets this buffer every x seconds, and appends it to the page. If your browser has aquired more then 512 lines it will start to remove the oldest lines until 512 lines remain.

If you now lower "loghistorylines" your browser will still display 512 lines, but because the oscam buffer is so small, you will miss many lines (for sure when your loglevel is > 0), and your log will be meaningless.

Attention: Log in status - High value extends the data is loaded in the web browser.
